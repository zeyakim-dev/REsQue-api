""""""""""""""""""""""""""""""""""""

기본 프로젝트 생성 (test_create_project)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
:시나리오:
    * 유효한 데이터로 프로젝트 생성
    * 생성된 프로젝트의 속성 검증

:검증 항목:
    * id가 유효한 UUID
    * title과 description이 일치
    * status가 ACTIVE
    * owner가 설정됨
    * members에 owner가 manager로 포함

잘못된 제목 길이 (test_create_project_with_invalid_title)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
:시나리오:
    * 빈 제목으로 생성 시도
    * 최대 길이 초과 제목으로 생성 시도

:검증 항목:
    * InvalidTitleError 예외 발생
    * 적절한 에러 메시지 포함

멤버 추가 (test_add_member)
~~~~~~~~~~~~~~~~~~~~~~~
:시나리오:
    * 활성 상태의 프로젝트에 멤버 추가
    * 다양한 역할로 멤버 추가

:검증 항목:
    * 멤버가 정상적으로 추가됨
    * 지정된 역할이 올바르게 설정됨
    * 중복 추가 시 예외 발생

초대 관리 (TestProjectInvitation)
"""""""""""""""""""""""""""""""

초대 생성 (test_create_invitation)
~~~~~~~~~~~~~~~~~~~~~~~~~
:시나리오:
    * 이메일로 멤버 초대
    * 초대 만료 시간 검증

:검증 항목:
    * 초대 객체가 생성됨
    * 만료 시간이 7일 후로 설정
    * 중복 초대 시 DuplicateInvitationError 발생

초대 수락 (test_invite_acceptance)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~
:시나리오:
    * 유효한 초대 코드로 수락 시도
    * 만료된 초대 코드로 수락 시도
    * 이미 수락된 초대 코드로 재시도

:검증 항목:
    * 유효한 경우 멤버로 추가되고 초대 상태가 ACCEPTED로 변경됨
    * 만료된 경우 ExpiredInvitationError 발생
    * 중복 수락 시 AlreadyAcceptedInvitationError 발생

수정 권한 확인 (test_can_modify)
~~~~~~~~~~~~~~~~~~~~~~~~~~
:시나리오:
    * 다양한 역할의 멤버로 수정 시도
    * 비멤버의 수정 시도

:검증 항목:
    * manager는 모든 수정 가능
    * member는 제한된 수정 가능
    * viewer는 수정 불가
    * 비멤버는 접근 불가

~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

테스트 설계 원칙
--------------

격리성
^^^^^
* 각 테스트는 독립적으로 실행
* fixture를 통한 테스트 데이터 관리
* 외부 의존성 최소화

가독성
^^^^^
* 명확한 시나리오 기술
* Given-When-Then 패턴
* 검증 항목 명시

커버리지
^^^^^^^
* 모든 도메인 규칙 검증
* 경계 조건 테스트
* 예외 케이스 포함
